# Contributing

## Правила работы
1. Новые изменения делай в `feat/*` ветках.
2. Коммиты — атомарные, по одной задаче.
3. Перед push:
   - `git status`
   - убедиться, что нет секретов/сессий
4. Для инфраструктурных изменений добавляй короткий runbook в описание PR.

## Стиль
- Python: простые rule-based скрипты, предсказуемый fallback без GPT.
- Конфиги: JSON в `config/`.
- Systemd: unit/timer в `infra/systemd-user/`.

## PR-чеклист
- [ ] Нет `.env`, `config.env`, `*.session`, ключей
- [ ] Обновлены связанные конфиги/доки
- [ ] Скрипт протестирован one-shot запуском
- [ ] Добавлен/обновлён отчёт о проверке (локально)

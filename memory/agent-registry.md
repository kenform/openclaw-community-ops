# Agent Registry

Статус: активная конфигурация (v1)
Обновлено: 2026-03-02

## ORCHESTRATOR (главный)
- Тип: управляющий контур
- Цель: планировать, делегировать, принимать результаты, собирать единый отчёт
- Зоны записи:
  - `memory/queue.md`
  - `memory/tasks-log.md`
  - `memory/daily-report-template.md`
- Не делает:
  - параллельную “ручную” работу за всех
  - рискованные внешние действия без явной команды

## RESEARCH AGENT
- Тип: аналитический исполнитель
- Цель: ресерч, выжимки, сравнения, гипотезы
- Зоны записи:
  - `vault/40 Resources/**`
  - аналитические секции в project notes
- Формат результата:
  - тезис
  - 3-7 ключевых фактов
  - риски/ограничения
  - следующий шаг

## BUILDER AGENT
- Тип: production/knowledge исполнитель
- Цель: создавать артефакты (шаблоны, md-файлы, структуры, инструкции)
- Зоны записи:
  - `vault/99 Templates/**`
  - `vault/20 Projects/**`
  - `vault/40 Resources/**`
- Формат результата:
  - что создано/изменено
  - точные пути
  - критерий готовности

---

## Общие правила
1. Один дирижёр: только ORCHESTRATOR управляет очередью.
2. Никаких конфликтов ownership: один файл — один ответственный в момент задачи.
3. Любая завершённая задача обязана иметь запись в `memory/tasks-log.md`.
4. Ночью только safe-mode задачи (без внешних публикаций/финопераций).
5. Одновременно активных задач: максимум 3.

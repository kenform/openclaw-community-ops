# Launch Kit: Урок №4 — OpenClaw: автоматизация рутины без потери контроля

## 1) Скрипт урока №4 (10–15 мин)

### Тема
Как использовать OpenClaw как личный операционный слой: меньше ручной рутины, больше времени на важное.

### Хук (0:00–0:30)
«Большинство задач не сложные — они повторяются. Сегодня покажу, как передать рутину ассистенту и не потерять контроль.»

### Суть (0:30–4:00)
Модель “Делегирование с рамками”:
1. Что можно автоматизировать безопасно
2. Что всегда оставляем с ручным подтверждением
3. Как задавать формат результата
4. Как проверять качество выполнения

**Шаблон операционного запроса:**
> Сделай [задача].
> Ограничения: [что нельзя делать без подтверждения].
> Формат отчёта: [кратко/структурно/с чеклистом].
> Если риск/неясность — остановись и спроси.

### Демо (4:00–10:00)
Кейс: ежедневный утренний блок
- проверка статуса сервисов
- короткий отчёт по критичным рискам
- список top-3 задач на день

Показываешь, что ассистент делает повторяемый цикл без “самодеятельности”.

### Финал (10:00–12:00)
«Автоматизация — это не хаос, а дисциплина: чёткая задача, рамки, формат отчёта и контроль.»

## 3) Домашка + чеклист

### Домашка
Выбери 3 повторяющиеся задачи и переведи их в делегируемый формат через OpenClaw.

### Что сдать
- список 3 задач
- шаблон запроса для каждой
- ограничения/стоп-правила
- формат отчёта

### Чеклист
- [ ] задачи действительно повторяющиеся
- [ ] для каждой задачи есть ограничения
- [ ] есть явный формат отчёта
- [ ] есть правило “остановиться и спросить при риске”
- [ ] после теста качество стало выше, а времени ушло меньше

---

## Бонус: что не делегировать без подтверждения
1. Публичные публикации/рассылки
2. Массовые удаления/редактирование данных
3. Изменения доступа/секретов/ключей
4. Любые действия с финансовыми последствиями

# Google Form E2E (вопросы подписчиков)

## Цель
Собирать вопросы подписчиков через форму и отвечать по структурированному потоку (без GPT fallback тоже работает).

## Структура Google Form
1. Имя / ник
2. Telegram username
3. Тема (выбор):
   - OpenClaw базовые
   - Настройка парсеров
   - Группы/контент
   - Ошибка/баг
   - Другое
4. Вопрос (длинный текст)
5. Уровень срочности (низкий/средний/высокий)
6. Согласие на обработку

## Логика обработки
- Все ответы идут в Google Sheets.
- Скрипт забирает новые строки, нормализует, классифицирует по теме и срочности.
- Формирует черновик ответа из шаблонов.
- Для high-priority — отдельный алерт в Telegram owner.

## Шаблоны ответов (база)
- Базовый: "Приняли вопрос, готовим ответ в рабочем окне." 
- Тех.ошибка: "Проверим логи/конфиг и вернёмся с конкретным решением." 
- Обучение: "Соберём короткий пошаговый план без воды."

## Что нужно от владельца
- URL формы
- URL связанной Google Sheets
- (Опц.) service-account JSON для автоматического чтения таблицы

# HEDGER — аналитика по собранным постам

## Краткий вывод
HEDGER в подаче ADEL выглядит как «алго-альтернатива» скучному рынку и неэффективному копитрейду. Контент-модель сильная (серийные отчёты + цифры + тизер), но пока недостаточно прозрачная по риск-метрикам и консистентности расчётов.

## Что выглядит сильным
1. Регулярные отчёты (Week 1/2) — повышают доверие и удержание.
2. Понятный narrative: «паркую ликву, тестирую инструмент, покажу результат».
3. Низкий порог входа («можно бесплатно потыкать»).
4. Подход “тест на дистанции” вместо мгновенных обещаний.

## Красные флаги / вопросы
1. Арифметические нестыковки в постах (нужно сверять net PnL по единой формуле).
2. Нет обязательного риск-блока: drawdown, комиссии, slippage, stop-режим.
3. Масштабирование с 9.5k к ~50k может разрушить текущую доходность.
4. Маркетинговый тон может перевешивать методологию.

## Что делать практично (если тестировать)
1. Запускать как пилот с лимитом риска, не как основную стратегию.
2. Вести trade-journal: вход/выход/объём/комиссия/причина.
3. Отдельно считать:
   - Gross PnL
   - Fees + slippage
   - Net PnL
   - Max drawdown
4. Установить hard-stop по просадке и лимит по дневному риску.

## Рамка принятия решения
- Подходит, если: есть дисциплина отчётности и верификация метрик.
- Не подходит, если: решения принимаются только по “процентам в посте”.

## Для твоего контента/монетизации
- Сильный формат: «Risk-first weekly review» вместо “хайпового профита”.
- Продукт: шаблон верифицируемого отчёта по стратегиям (для каналов/команд).

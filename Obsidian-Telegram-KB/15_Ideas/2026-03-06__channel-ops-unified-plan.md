# Unified Channel Ops Plan (from voice notes)

## Цель
Собрать все текущие голосовые/текстовые заметки в единый операционный план по каналам и автопостингу.

## Приоритеты
1. Довести читаемый и стабильный формат автопостинга (без поломки оформления).
2. Создать отдельный канал **OpenClaw News**.
3. Собрать и поддерживать папку/лист годных источников по OpenClaw для парсинга.
4. Унифицировать нижний блок ссылок во всех каналах и шаблонах постов.

## Конкретные задачи
- [ ] Создать канал OpenClaw News.
- [ ] Подключить отдельный pipeline парсинга новостей по OpenClaw/агентам.
- [ ] Сформировать curated-список Telegram-источников по OpenClaw.
- [ ] Внедрить единый шаблон ссылок внизу поста: Биржи / Канал / Чат.
- [ ] Провести ревизию старых постов и исправить ссылки/формат там, где нужно.

## Тех. заметки
- Учитывать Telegram FloodWait и формат fallback.
- Сохранять единый стиль постов (заголовок, выжимка, вывод, ссылки, теги).
- Не публиковать низкосигнальный или технический мусор.

## Инфраструктурная заметка (2026-03-07)

### Текущий сервер (2 CPU / 4 GB / ~20 GB free)
- Не разворачивать полный стек 7 агентов на постоянной основе.
- Практичный режим: оставить основной контур и добавить только `мама-агента` (always-on, приоритет P0).
- Остальные агенты — по требованию/окнами, не параллелить тяжёлые задачи.

### Новый сервер (рекомендуемая цель)
- Полный мультиагентный стек (7 ролей) разворачивать уже на новом сервере.
- Минимум под рост: 6 vCPU / 12 GB RAM / 120+ GB NVMe.
- Комфортно: 8 vCPU / 16 GB RAM / 160–240 GB NVMe.

### Миграция (черновик плана)
1. На старом сервере остановить сервисы и зафиксировать состояние.
2. Перенести данные (`~/.openclaw`, `userbot`, workspace, env/session/config) на новый.
3. На новом сервере поднять те же версии runtime и поэтапно запустить сервисы.
4. Пройти smoke-check: status, тест-пост, проверка always-on mama-agent.
5. Только после проверки выключать старый сервер.

## Схема 7 агентов (зафиксировано)

### Роли
0. Оркестратор — маршрутизация задач, приоритеты, контроль очередей.
1. Dev-агент — только разработка (сайты, фиксы, PR-цикл).
2. Telegram Ops-агент — парсинг/постинг/формат/ссылки/анти-дубли.
3. YouTube-агент — контент-план, SEO, упаковка роликов.
4. Мама-агент (always-on) — отдельный стабильный контур помощи.
5. OpenClaw News-агент — отдельный канал + curated-источники по OpenClaw.
6. English-агент — практика английского и прогресс.
7. Study Super-агент — PDF/DOCX/PPT/изображения/голосовые → структурированные выжимки.

### Приоритеты
1. P0: Мама-агент
2. P1: Telegram Ops + Dev
3. P2: Study + OpenClaw News
4. P3: YouTube + English

### Режим работы
- Always-on: Оркестратор, Мама-агент, Telegram Ops.
- On-demand/по расписанию: Dev, YouTube, OpenClaw News, English, Study.

### Ресурсы
- Старт: 4 vCPU / 8 GB / 120 GB NVMe / swap 8 GB.
- Комфорт: 8 vCPU / 16 GB / 200+ GB NVMe / swap 16 GB.

### Операционные правила
- Ограничение параллелизма тяжёлых задач.
- Раздельные очереди по доменам (mom/telegram/dev/study/media).
- Лимиты по времени задач + авто-повтор после FloodWait.
- Единый лог и алерты по падениям.

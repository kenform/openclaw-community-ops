# Разбор поста: Итоги недели по HEDGER

## Источник
- Канал: ADEL | BLOG (@adel_crypta)
- Формат: weekly-перформанс отчёт + CTA в комментарии + реферальные ссылки

## Суть
Автор показывает публичную доходность за неделю в HEDGER:
- старт: $9,500
- текущий баланс: $9,974
- заявленный weekly-профит: $212
- далее добавлен капитал +$38k (ожидание отчёта на следующей неделе)

Основной месседж: стратегия масштабируется, доходность выглядит «мегахорошей», ждём стресс-тест на крупном объёме.

## Что сильного в посте
1. Конкретные цифры (депо/баланс/период) — повышают доверие.
2. Серийный формат «неделя 1/2/3» — удерживает аудиторию.
3. Призыв к вопросам по гайду — снижает барьер новичкам.
4. Мостик на следующий отчёт — создаёт ожидание.

## Слабые места / риски
1. Несостыковка цифр: 9,500 → 9,974 даёт +474, а в тексте +212 (нужно пояснение, иначе падает доверие).
2. Нет разбивки риска: просадка, комиссии, ликвидность, проскальзывание.
3. Сильный акцент на реф-ссылки может выглядеть как шиллинг без прозрачной методики.
4. Масштабирование до ~50k может резко изменить доходность (стаканы/ликвидность/исполнение).

## Что взять тебе в работу
- Делать weekly-отчёты с обязательным блоком «методика и риски», не только «профит».
- Всегда синхронизировать математику в посте (P&L = баланс - депозит ± ввод/вывод).
- Добавлять мини-таблицу: депозит, текущий баланс, net PnL, комиссии, max drawdown.
- Делать честный пост-мортем: где доход, где риск, где границы масштабирования.

## Action items (для твоего контента)
1. Подготовить шаблон «прозрачный weekly-report» (цифры + риск + вывод).
2. Добавить блок «что может пойти не так» в каждый отчёт.
3. Показать 2 сценария: базовый и стресс (рост объёма x3/x5).
4. Разделить “образовательный контент” и “реферальные ссылки”, чтобы не терять доверие.

## Потенциал дохода
1. Платный формат «риск-разбор стратегий» (а не сигналы): подписка/закрытый разбор.
2. Консалтинг по структуре публичных отчётов для крипто-каналов.
3. Собственный продукт: «Risk-first шаблоны для отчётов/портфелей».
4. Партнёрки только с прозрачным дисклеймером + KPI эффективности.

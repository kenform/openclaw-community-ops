# Статусы и команды

## Проверка статуса (все ключевые сервисы)
```bash
systemctl --user is-active telegram-pipeline-v1.service
systemctl --user is-active userbot.service
systemctl --user is-active userbot2.service
systemctl --user is-active telegram-obsidian.service
```

## Подробный статус
```bash
systemctl --user status telegram-pipeline-v1.service
systemctl --user status userbot.service
systemctl --user status userbot2.service
systemctl --user status telegram-obsidian.service
```

## Логи (последние строки)
```bash
journalctl --user -u telegram-pipeline-v1.service -n 50 --no-pager
journalctl --user -u userbot.service -n 50 --no-pager
journalctl --user -u userbot2.service -n 50 --no-pager
journalctl --user -u telegram-obsidian.service -n 50 --no-pager
```

## Перезапуск
```bash
systemctl --user restart telegram-pipeline-v1.service
systemctl --user restart userbot.service
systemctl --user restart userbot2.service
systemctl --user restart telegram-obsidian.service
```

## Быстрая диагностика «не постит»
1. Проверить `is-active`.
2. Проверить последние логи `journalctl`.
3. Проверить целевой канал/права публикации.
4. Для OpenClaw parser (`userbot2`) — проверить SAFE MODE изменения и тестовый `run_digest`.
